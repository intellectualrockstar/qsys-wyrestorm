Wyrestorm NHD

Use this plugin to discover all Wyrestorm NetworkHD transmitter and receiver devices on the network, assign aliases, and route Tx to Rx for an NHD device.

Note: See the Wyrestorm website for product information and documentation.

Compatibility

The plugin supports the NHD 100, 200, 400, and 600 series models. The plugin was developed and tested with:

  • Q-SYS Designer version 8.4.0
  • Wyrestorm API version 1.21
  • Wyrestorm system version 8.3.1 (8.3.8)

Connecting to the Device

To begin using the plugin, drag it into the schematic and configure its Properties. Then, press F5 to save your design to the Core and run it.

In the plugin's Setup tab:

  1. Type the IP Address, Port, and Password for the NHD-000-CTL.
  2. Press Enter.

The plugin will automatically attempt a connection. If you see "OK" status, you are successfully connected to the device. If you see a "Fault" error, check to make sure you entered the correct parameters.

Properties

Is Managed

Select 'Yes' to add the plugin to your Inventory list, which makes the plugin's Status available for monitoring in Core Manager and Enterprise Manager. When enabled, you can then specify a Name and Location for the plugin, as with any other Inventory item.

For more information, refer to the Plugins topic in the Q-SYS Help.

Debug Print

When Show Debug is set to 'Yes', configure how debug print statements are shown:

  • None: (Default) Disables debug print statements.
  • Tx/Rx: Prints sent and received data.
  • Tx: Prints only sent data.
  • Rx: Prints only received data.
  • Function Calls: Prints the name of functions as they are called.
  • All: Enables all debug print statements.

Tx Count

Set the total number of transmitters in the system, from 1 to 128 (default is 1).

Rx Count

Set the total number of receivers in the system, from 1 to 128 (default is 1).

Show Debug

Select 'Yes' to show the Debug Output window. Use the Debug Print property to configure debug print statements.

For more information, see the Debug Output topic in the Q-SYS Help.

Controls

Setup

IP Address, Port, Password

Specify these parameters as configured on the device.

API Version, System Version

These read-only fields populate once the plugin successfully connects to the device.

Refresh Devices

Click to manually re-discover all NHD devices on the network. The plugin does this automatically whenever a device leaves or joins the network.

Connection Status

Displays the current connection status:

  • OK: Plugin is connected to the unit
  • FAULT: Check the credentials to connect to the device.
  • MISSING: No device is found at the specified IP address.

Tx 1 - n

Name

The number of transmitters is determined by the Tx Count property. Click the Name box to select a transmitter.

Reboot

Click to reboot the selected transmitter.

HDCP

LED indicates if HDCP is enabled for the selected transmitter.

Online

LED indicates if transmitter is online.

Rx 1 - n

Name

The number of receivers is determined by the Rx Count property. Click the Name box to select a receiver.

Source

Displays the current transmitter routed to that receiver as well as a drop down list of all transmitters able to be routed to that receiver. Simply select a transmitter from the drop down to route it to that receiver.

Reboot

Click to reboot the selected receiver.

Online

LED indicates if selected receiver is online.
